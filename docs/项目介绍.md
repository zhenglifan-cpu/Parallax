## 一、游戏概述

**同途 Parallax** 是一款双人在线协作平台跳跃游戏。两位玩家分别扮演"红鸟"与"黑鸟"，在同一张地图上看到截然不同的世界景象——红鸟眼中是温暖明亮的庆典世界，黑鸟眼中是阴冷黑暗的末日世界。玩家需要协作解谜、通过关卡，共同推进叙事。

### 核心特色

| 特色 | 说明 |
|------|------|
| **双重世界观** | 同一地图在两位玩家屏幕上呈现完全不同的视觉风格与色彩 |
| **在线协作** | 基于持久世界（Persistent World）的双人实时联机 |
| **内置地图编辑器** | 开发者可在游戏内实时编辑关卡并自动同步 |
| **Celeste 风格物理** | 土狼时间、跳跃缓冲、长短跳区分等精细操控手感 |
| **NanoVG 矢量渲染** | 全部画面通过 NanoVG 矢量图形管线绘制，风格统一 |

## 三、联机方案

### 3.1 网络架构

游戏采用 **持久世界（Persistent World）** 模式，即服务器持续运行，玩家随进随出。

```
配置项（.project/settings.json）:
  multiplayer.enabled = true
  multiplayer.max_players = 2
  multiplayer.persistent_world.enabled = true
```

- **入口分离**: 客户端入口 `Network/Client.lua`，服务端入口 `Network/Server.lua`
- **权威服务器**: 所有游戏逻辑由服务端判定，客户端发送控制输入，服务端广播状态

### 3.2 客户端-服务端通信

基于远程事件（Remote Events）实现双向通信：

**客户端 → 服务端事件**:

| 事件名 | 用途 |
|--------|------|
| `CLIENT_READY` | 客户端就绪通知 |
| `PAIR_REQUEST` | 配对请求 |
| `UNPAIR_REQUEST` | 解除配对 |
| `CHARACTER_PICK` | 选择角色（红鸟/黑鸟） |
| `DEBUG_SOLO` | 单人调试模式 |
| `DEBUG_RELOAD` | 编辑器同步重载地图 |
| `SPAWN_TELEPORT` | 传送到出生点 |

**服务端 → 客户端事件**:

| 事件名 | 用途 |
|--------|------|
| `PLAYER_INFO` | 玩家信息同步 |
| `PAIR_RESULT` | 配对结果 |
| `PARTNER_LEFT` | 搭档断开通知 |
| `ENTER_SELECT` | 进入角色选择阶段 |
| `CHARACTER_RESULT` | 角色选择结果 |
| `ENTER_GAME` | 进入游戏 |
| `GAME_EVENT` | 游戏内事件广播 |
| `SCENE_CHANGE` | 场景切换 |

### 3.3 输入同步

使用**控制位（Control Bits）**压缩传输玩家输入：

```lua
CTRL_MOVE_LEFT  = 1   -- 左移
CTRL_MOVE_RIGHT = 2   -- 右移
CTRL_JUMP       = 4   -- 跳跃
CTRL_INTERACT   = 8   -- 交互
CTRL_PUSH       = 16  -- 推动
```

客户端每帧将按键状态压缩为单个整数发送至服务端，服务端解包后驱动角色物理。

### 3.4 好友码匹配

通过 `clientCloud` API 实现无需第三方服务的房间匹配：

1. **创建房间**: 生成 6 位好友码（排除易混淆字符），写入云变量 `room_XXXXXX = timestamp`
2. **查找房间**: 通过 `GetRankList()` 扫描所有活跃房间
3. **加入房间**: 匹配到相同好友码的两位玩家配对入场

状态机流程: `IDLE → CREATING_ROOM → WAITING → JOINING_ROOM → QUEUING → CONNECTED`

字符集: `ABCDEFGHJKMNPQRSTUVWXYZ23456789`（排除了 I/L/O/0/1 等易混淆字符）

## 四、游戏画面渲染

### 4.1 渲染管线概览

全部游戏画面通过 **NanoVG** 矢量图形 API 绘制，不使用传统 3D 网格。渲染流程如下：

```
每帧渲染顺序:
  1. 天空渐变背景
  2. 视差滚动层（山脉、云层等图片）
  3. 平台/地面（砖块纹理）
  4. 桥梁
  5. 梯子
  6. 水面（含波浪动画）
  7. 箱子（含物理旋转）
  8. 触发器/传送门
  9. NPC
  10. 玩家角色（精灵帧动画）
  11. 地面覆盖层（Overlay 装饰图）
  12. 碰撞调试可视化（可选）
```

### 4.2 坐标变换

世界坐标到屏幕坐标的核心变换：

```lua
function WorldToScreen(wx, wy, camX, camY, screenW, screenH)
    local sx = (wx - camX) * PPU + screenW * 0.5
    local sy = screenH * 0.5 - (wy - camY) * PPU
    return sx, sy
end
```

- **PPU (Pixels Per Unit)**: 90 像素/单元格
- **CELL_SIZE**: 0.5 米
- **设计分辨率**: 960×540

### 4.3 双世界配色系统（DualWorldSkin）

同一地图在红鸟和黑鸟视角下呈现完全不同的视觉风格：

| 元素 | 红鸟世界（庆典） | 黑鸟世界（末日） |
|------|----------------|----------------|
| 天空 | 明亮暖色渐变 | 深蓝灰暗渐变 |
| 山脉 | 暖橙/棕色层次 | 冷蓝/深灰层次 |
| 地面 | 温暖红棕砖块 | 暗灰石质表面 |
| 水面 | 清澈蓝绿 | 暗沉深蓝 |
| NPC | 明亮友好色调 | 阴暗警示色调 |
| 粒子 | 金色/暖光 | 冷白/灰烬 |

配色通过 `DualWorldSkin.Get(role)` 获取对应角色的完整调色板，渲染器统一使用调色板数据绘制。

### 4.4 视差背景

多层视差滚动背景，每层有独立的滚动速率：

- **天空渐变层**: 固定背景，使用线性渐变填充
- **图片视差层**: 支持多张图片以不同视差因子水平无缝拼贴滚动
- **程序化山脉层**: 使用调色板颜色绘制多层远山剪影

### 4.5 平台渲染

平台使用**砖块纹理**风格绘制：

- 渐变填充底色
- 灰缝线（mortar lines）模拟砖墙纹理
- 顶部高光条
- 支持坡道（slope）的三角形绘制
- 覆盖层（Overlay）可在平台上方叠加装饰图片，并自动隐藏被覆盖的砖块

### 4.6 精灵帧动画（SpriteAnim）

角色动画基于 **JSON 定义的精灵表**：

```
数据格式:
  - PNG 精灵表图片（单行排列的帧）
  - JSON 帧数据（每帧宽高、时长）

支持动画:
  - idle（待机）
  - run（奔跑，由水平速度驱动帧选择）
  - jump（跳跃，分上升/下落阶段）
```

关键设计: `DrawAtTime(cx, cy, drawH, flipX, elapsed)` 方法支持按外部时间戳绘制，实现每位玩家独立的动画时间轴（联机时各客户端独立计时）。

### 4.7 水面效果

水面使用正弦波动画模拟水波：

- 上部波浪线随时间变化
- 半透明填充
- 波光粒子效果

## 五、内置地图编辑器

### 5.1 概述

游戏内置完整的地图编辑器，在 `debugSolo` 模式下按 **T 键** 激活。编辑器支持所有关卡元素的创建和编辑，并能实时同步到服务端。

### 5.2 编辑工具

| 工具 | 功能 | 操作方式 |
|------|------|---------|
| **Ground** | 绘制/擦除地面格 | 鼠标绘制，支持画笔大小调节 |
| **Slope** | 放置坡道 | 选择方向后点击放置 |
| **NPC** | 放置/编辑 NPC | 点击放置，可编辑对话内容 |
| **Portal** | 放置传送门 | 设置目标场景/坐标 |
| **Spawn** | 设置出生点 | 区分红鸟/黑鸟出生位置 |
| **Water** | 绘制水域 | 定义水面矩形区域 |
| **Bridge** | 放置桥梁 | 定义起止坐标 |
| **Ladder** | 放置梯子 | 定义起止坐标 |
| **Crate** | 放置可推动箱子 | 点击放置 |
| **Move** | 移动已有元素 | 拖拽移动 |

### 5.3 视图模式

编辑器支持三种视图切换：

- **Logic 视图**: 显示逻辑网格与碰撞区域
- **Red Bird 视图**: 预览红鸟世界的视觉效果
- **Black Bird 视图**: 预览黑鸟世界的视觉效果

### 5.4 自由覆盖层系统（Free Overlay）

除了基于网格的地形编辑，编辑器还支持**自由覆盖层**功能：

- **资源浏览器**: 从静态资源清单中浏览可用的装饰图片
- **自由放置**: 装饰图不受网格约束，可放置在任意位置
- **变换操作**: 支持拖拽移动、旋转、缩放
- **复制粘贴**: 快速复制已有装饰元素
- **遮挡处理**: 覆盖层图片会自动隐藏下方的砖块纹理

### 5.5 撤销/重做

编辑器实现了完整的撤销/重做栈，所有编辑操作均可回退。

### 5.6 自动同步

编辑器修改后自动同步到服务端：

- **防抖延迟**: 0.5 秒（避免频繁同步）
- **同步流程**: 标记脏数据 → 等待防抖 → 序列化地图 → 通过 `DEBUG_RELOAD` 事件通知服务端重载
- **热重载**: 服务端收到重载事件后立即加载新地图数据，无需重启

### 5.7 场景/幕管理

- 支持多场景（Scene）、多幕（Act）的层级关卡结构
- 编辑器内可切换当前编辑的场景
- 所有关卡均通过编辑器创建，无硬编码关卡数据

## 六、物理系统

### 6.1 参数配置

游戏采用 **Celeste 风格**的精细平台跳跃手感：

| 参数 | 值 | 说明 |
|------|-----|------|
| GRAVITY | 30 m/s² | 重力加速度（高于真实值，手感更紧凑） |
| PLAYER_SPEED | 2.0 m/s | 玩家水平移动速度 |
| JUMP_SPEED | 14.0 m/s | 跳跃初速度 |
| COYOTE_TIME | 0.083 秒 | 离开平台后仍可跳跃的宽容时间 |
| JUMP_BUFFER | 0.083 秒 | 落地前提前按跳跃的缓冲时间 |
| AIR_JUMPS | 1 次 | 空中额外跳跃次数（二段跳） |
| LONG_JUMP_HOLD | 0.3 秒 | 长按跳跃键的最大持续时间 |
| CUT_VELOCITY | 8.4 m/s | 松开跳跃键后的速度截断值 |

### 6.2 操控机制

- **长短跳区分**: 按住跳跃键时间越长跳得越高，松开后速度被截断
- **土狼时间（Coyote Time）**: 离开平台边缘后短时间内仍可起跳
- **跳跃缓冲（Jump Buffer）**: 落地前按下跳跃键，落地瞬间自动起跳
- **二段跳**: 空中可再跳一次
- **坡道物理**: 支持在坡面上行走和滑行

### 6.3 可交互物体

| 物体 | 物理行为 |
|------|---------|
| 箱子（Crate） | 可推动，受重力影响，有旋转 |
| 桥梁（Bridge） | 静态碰撞体，玩家可行走其上 |
| 梯子（Ladder） | 触发区域，进入后切换为攀爬模式 |
| 水域（Water） | 触发区域，进入后切换浮力物理 |

## 七、角色与剧情系统

### 7.1 角色选择

游戏开始前，两位玩家通过**卡片式角色选择界面**选择扮演红鸟或黑鸟：

- 每张卡片展示角色预览与配色
- 冲突检测：若两人选择相同角色则提示重选
- 选择结果通过 `CHARACTER_PICK` / `CHARACTER_RESULT` 事件同步

### 7.2 角色常量

```lua
ROLE_RED   = "red"    -- 红鸟（庆典世界视角）
ROLE_BLACK = "black"  -- 黑鸟（末日世界视角）
```

### 7.3 NPC 对话系统

NPC 对话根据玩家角色呈现不同内容：

- **红鸟视角**: 对话内容偏向庆典、冒险、温暖
- **黑鸟视角**: 对话内容偏向末日、生存、警示

对话数据存储在 `LevelData` 模块中，按场景和 NPC 索引管理。

## 九、资源管理

### 9.1 资源类型

| 类型 | 目录 | 格式 |
|------|------|------|
| 精灵表 | assets/Sprites/ | PNG + JSON（帧数据） |
| 背景图 | assets/Textures/ | PNG |
| 覆盖层装饰 | assets/Overlays/ | PNG |
| 音效 | assets/Sounds/ | OGG |
| 字体 | assets/Fonts/ | TTF |

### 9.2 覆盖层资源清单

编辑器的装饰图系统基于静态 `manifest.json` 管理可用资源列表，编辑器的资源浏览器从中读取并展示可放置的装饰图片。
