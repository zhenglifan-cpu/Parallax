# 《Parallax》游戏策划案 (MVP)

> 复刻《不同的冬天/BOKURA》核心玩法 | 双人联机 | 同关卡异画面 | 2D 横版解谜平台跳跃

---

## 1. 核心概念

**一句话描述**: 两名玩家在同一关卡中并肩登山，但一人经历的是追寻荣耀的冒险，另一人经历的是逃离灾难的求生。双方必须通过语音沟通，利用各自世界中的信息差来合作解谜。

**与 BOKURA 的对应关系**:

| BOKURA | 本作《同途 / Parallax》 |
|--------|------------------------|
| 动物世界 (小绿) | **冒险世界** (暖色、明亮、主动探索 → 红鸟) |
| 机械世界 (小蓝) | **求生世界** (冷色、阴暗、被动逃离 → 黑鸟) |
| 两个离家出走的男孩 | 红鸟 (追寻勇者徽章) + 黑鸟 (逃往山顶庇护所) |
| 看到鹿的尸体后世界分裂 | 开场：两只鸟在山脚醒来，各自收到不同的信件 |

## 2. 联机架构

### 2.1 联机方式 (复刻 BOKURA)

```
┌─────────────────────────────────────────┐
│            UrhoX 服务器                  │
│  ┌───────────────────────────────────┐  │
│  │        房间 (Room Code)            │  │
│  │  - 同步两名玩家的位置/状态/操作    │  │
│  │  - 转发对方操作结果                │  │
│  │  - 不同步画面渲染(各端本地渲染)    │  │
│  └───────────────────────────────────┘  │
│       ↑                      ↑          │
│    玩家A                   玩家B        │
│  (乐观世界)              (悲观世界)      │
│  本地渲染乐观层           本地渲染悲观层  │
└─────────────────────────────────────────┘
```

**匹配流程 (复刻 BOKURA 房间码机制)**:
1. 玩家 A 创建房间 → 获得 6 位房间码
2. 玩家 B 输入房间码 → 加入房间
3. 服务器自动分配角色: 房主 = 乐观方 (红鸟)，加入者 = 悲观方 (黑鸟)
4. 双方确认 → 游戏开始

### 2.2 同步策略

**需要同步的数据**:
| 数据类型 | 同步方式 | 说明 |
|---------|---------|------|
| 玩家位置 (x, y) | 每帧广播 | 两端共享同一逻辑坐标系 |
| 玩家动作状态 | 事件触发 | 跳跃、推箱子、攀爬、站立/行走 |
| 机关状态 | 事件触发 | 开关、门、平台的开/关 |
| 谜题进度 | 事件触发 | 完成某个谜题节点 |
| 检查点 | 事件触发 | 触达存档点时双方同步记录 |

**不需要同步的数据**:
- 画面渲染层 (乐观/悲观的视觉差异完全在各自客户端本地处理)
- 视差背景滚动
- 粒子/光效等纯视觉效果

## 3. 双世界画面系统 (核心机制)

### 3.1 设计原则

同一关卡地图有一份**逻辑数据** (碰撞、位置、机关) + 两份**视觉皮肤** (乐观/悲观)。

```
逻辑层 (共享):
┌────────────────────────────────────────────────┐
│ 碰撞体  平台  机关触发区  NPC位置  物品位置      │
└────────────────────────────────────────────────┘

视觉层 A - 乐观世界 (红鸟看到的):
┌────────────────────────────────────────────────┐
│ 暖色调背景  繁花似锦  阳光明媚  友善的NPC外观    │
│ 同一个机关 → 看起来是"花朵开关"                  │
│ 同一个障碍 → 看起来是"藤蔓桥"                    │
└────────────────────────────────────────────────┘

视觉层 B - 悲观世界 (黑鸟看到的):
┌────────────────────────────────────────────────┐
│ 冷色调背景  枯枝残叶  浓雾弥漫  扭曲的NPC外观    │
│ 同一个机关 → 看起来是"骨头拉杆"                  │
│ 同一个障碍 → 看起来是"断裂的锁链"                │
└────────────────────────────────────────────────┘
```

### 3.2 视觉差异映射表 (MVP 第一关)

| 逻辑元素 | 乐观世界 (红鸟) | 悲观世界 (黑鸟) |
|---------|----------------|----------------|
| 地面平台 | 铺满花草的石板路 | 开裂的泥土地面 |
| 背景远景 | 蓝天白云、远山 (参考图6科幻塔城的层次感，但用暖色自然风) | 浓雾、枯树剪影 (参考图5 Limbo 风格) |
| 背景中景 | 郁郁葱葱的树木、桥梁 | 残破建筑剪影、铁丝网 |
| 背景近景 | 花丛、蝴蝶 | 荆棘、乌鸦 |
| 可推动方块 | 带花纹的木箱 | 生锈铁笼 |
| 开关机关 | 发光的花蕊按钮 | 嵌在墙上的暗红色拉杆 |
| 门/通道 | 花藤拱门 | 铁栅栏门 |
| NPC (路障) | 沉睡的大猫 (友善) | 黑影怪物 (恐怖) |
| 对方角色 | 看到黑鸟 → 显示为灰色鸟 | 看到红鸟 → 显示为灰色鸟 |

### 3.3 关键: 信息差驱动解谜

复刻 BOKURA 的核心——**同一个物体在两个世界中表现不同，导致只有一方能看到关键信息**:

**示例谜题: "隐形路径"**
- 乐观方看到: 前方有一段完整的花藤桥
- 悲观方看到: 前方是断崖，没有桥
- **实际逻辑**: 桥存在，但只在乐观方画面可见。悲观方必须信任乐观方的描述，闭眼走过去
- **解法**: 乐观方通过语音告诉悲观方"往前走，有桥的"

**示例谜题: "单向可见开关"**
- 乐观方看到: 墙上有一朵发光的花，但无法互动
- 悲观方看到: 墙上有一个可拉动的拉杆
- **实际逻辑**: 只有悲观方能操作这个开关，但开关打开的门只在乐观方的画面中可见路径提示 (门后的正确走法)
- **解法**: 悲观方拉下拉杆 → 乐观方告诉悲观方门后该怎么走

## 4. 视差背景系统

### 4.1 层次结构

每个世界有 5 层视差背景，从远到近:

```
Layer 0: 天空/基底色    滚动系数: 0.0 (固定不动)
Layer 1: 远景          滚动系数: 0.1 (极缓移动)
Layer 2: 中远景        滚动系数: 0.3
Layer 3: 中近景        滚动系数: 0.6
Layer 4: 前景装饰      滚动系数: 1.2 (比主角快，遮挡效果)
─── 主角层 ───         滚动系数: 1.0 (跟随相机)
```

### 4.2 乐观世界视差层设计 (参考图1/图2风格)

| 层级 | 内容 | 色调 |
|------|------|------|
| Layer 0 | 渐变天空 (淡蓝→橙粉，参考图2的晚霞感) | #87CEEB → #FFB6C1 |
| Layer 1 | 远山轮廓 + 淡云 (参考图2的山脉剪影) | 淡紫/淡蓝半透明 |
| Layer 2 | 松树林/建筑轮廓 (参考图2的森林层) | 深青绿 |
| Layer 3 | 近处树木、花丛、桥梁细节 | 翠绿/暖黄 |
| Layer 4 | 前景树叶、飘落花瓣 | 深绿/粉色 |

### 4.3 悲观世界视差层设计 (参考图1/图5/图6风格)

| 层级 | 内容 | 色调 |
|------|------|------|
| Layer 0 | 深色渐变 (深灰→暗紫) | #1a1a2e → #2d2d44 |
| Layer 1 | 远处工业剪影/荒山 (参考图6的科幻塔城轮廓) | 灰蓝半透明 |
| Layer 2 | 废弃建筑群/铁塔 (参考图1的城市缝隙感) | 深蓝灰 |
| Layer 3 | 枯树、破碎栅栏、电线杆 (参考图5的Limbo氛围) | 暗灰/暗棕 |
| Layer 4 | 前景荆棘、飘落灰烬、雾气 | 黑色/灰色半透明 |

### 4.4 技术实现 (UrhoX)

使用 UrhoX 的 2D 精灵系统实现视差:

```lua
-- 每层是一个 Node，通过 Z 轴深度排序
-- 在 Update 中根据相机移动量 × 滚动系数 偏移每层位置

local parallaxLayers = {
    { node = bgLayer0, factor = 0.0 },   -- 天空固定
    { node = bgLayer1, factor = 0.1 },   -- 远景
    { node = bgLayer2, factor = 0.3 },   -- 中远景
    { node = bgLayer3, factor = 0.6 },   -- 中近景
    { node = fgLayer4, factor = 1.2 },   -- 前景
}

function HandleUpdate(eventType, eventData)
    local cameraDelta = camera:GetPosition() - lastCameraPos
    for _, layer in ipairs(parallaxLayers) do
        layer.node:SetPosition2D(
            layer.node.position2D + Vector2(cameraDelta.x * layer.factor, 0)
        )
    end
    lastCameraPos = camera:GetPosition()
end
```

## 6. MVP 关卡设计 (第一关: 到达第一个存档点)

### 6.1 关卡流程 (线性，约 5-8 分钟)

```
[起点] → [教学区] → [谜题1] → [谜题2] → [谜题3] → [存档点/MVP终点]
```

### 6.2 教学区: 基础操作

**目的**: 让两名玩家学会移动、跳跃、互动，并意识到"我们看到的不一样"。

**布局**:
```
两只鸟同时出现在一片平地上，各自屏幕显示不同的背景。
前方有一个 NPC 站在路中间。
```

**乐观方看到**: 一只友善的蝴蝶停在路中间，旁边有文字提示"按 E 打招呼"
**悲观方看到**: 一团黑雾挡在路中间，旁边有文字提示"按 E 驱散"

**两人都按 E 后**: NPC 消失，路径打开。双方第一次意识到"我们看到的东西不一样"。

### 6.3 谜题 1: 隐形平台

**核心机制**: 信息差——只有一方能看到路径。

```
地形 (逻辑层):
                    ┌──┐
                    │终│
          ┌─┐ ┌─┐  │点│
     ┌─┐  │ │ │ │  │  │
     │ │  │ │ │ │  │  │
─────┘ └──┘ └─┘ └──┘  └────

```

**乐观方看到**: 所有平台完整可见，像一组花藤编成的踏板
**悲观方看到**: 只能看到起点和终点平台，中间一片虚空

**解法**: 乐观方通过语音指导悲观方: "往右跳一格""再往上跳"。悲观方必须信任对方的描述盲跳。

### 6.4 谜题 2: 单向开关与门

**核心机制**: 非对称操作——一方操作机关，另一方获得通路。

```
地形 (逻辑层):
                  ┌─────────┐
     [开关A]      │  门(关)   │    [开关B]
─────────────────┤          ├───────────────
                  └─────────┘
     (悲观方可见)              (乐观方可见)
```

**乐观方看到**: 左侧墙壁光滑 (看不到开关A)，右侧有一朵发光花 (开关B)
**悲观方看到**: 左侧有一个暗红拉杆 (开关A)，右侧墙壁光滑 (看不到开关B)

**解法**:
1. 悲观方描述"我左边有个拉杆" → 拉下开关 A → 门打开一半
2. 乐观方描述"我右边有朵花" → 按下开关 B → 门完全打开
3. 两人通过门继续前进

### 6.5 谜题 3: 协作推箱子

**核心机制**: 同一个物体在两个世界中有不同的物理表现。

```
地形 (逻辑层):
                 ┌─────┐
                 │ 高台 │
     [方块]      │     │
─────────────────┴─────┴────────
```

**乐观方看到**: 方块是一个轻巧的木箱，可以推动，画面中高台旁有一个花环标记 (提示箱子应该放在这里)
**悲观方看到**: 方块是一个沉重的铁笼，无法推动 (按 E 提示"太重了")，画面中高台上有一把钥匙的轮廓

**解法**:
1. 乐观方推方块到高台下方
2. 悲观方站在方块上 → 跳上高台
3. 高台上有一个只有悲观方能操作的机关 → 触发后打开前方道路

### 6.6 存档点 (MVP 终点)

两只鸟走到一个特殊区域:
- **乐观方看到**: 一棵发光的大树，树下有暖光
- **悲观方看到**: 一盏孤独的路灯，灯下有微弱光圈

两人同时站在光圈内 → 触发存档 → MVP 结束画面。

### 8.2 核心渲染逻辑

```lua
-- WorldRenderer.lua 核心伪代码

function WorldRenderer:LoadLevel(levelData, playerRole)
    -- playerRole: "optimist" 或 "pessimist"
    self.role = playerRole

    -- 1. 加载逻辑层 (两端一致)
    for _, plat in ipairs(levelData.platforms) do
        self:CreateCollisionPlatform(plat)  -- 物理碰撞体，双方共享
    end

    -- 2. 加载视觉层 (根据角色不同)
    for _, plat in ipairs(levelData.platforms) do
        local visual = plat.visual[self.role]
        if visual then
            self:CreateVisualSprite(plat, visual)  -- 只渲染本方的皮肤
        end
    end

    -- 3. 加载可交互物体 (根据 visible_to 过滤)
    for _, obj in ipairs(levelData.interactables) do
        local visual = obj.visual[self.role]
        if visual then
            self:CreateInteractable(obj, visual)
        end
        -- 即使不可见，仍注册逻辑 (用于接收对方的操作结果)
        self:RegisterInteractableLogic(obj)
    end

    -- 4. 加载视差背景
    local bgLayers = levelData.parallax_bg[self.role]
    self.parallaxMgr:Setup(bgLayers)
end
```

## 9. MVP 验收标准

| 编号 | 验收项 | 标准 |
|------|--------|------|
| 1 | 联机连接 | 通过房间码成功配对，延迟 < 200ms |
| 2 | 角色分配 | 房主=乐观方(红鸟)，加入者=悲观方(黑鸟) |
| 3 | 异世界渲染 | 同一关卡，两端画面风格完全不同 |
| 4 | 视差背景 | 5 层视差滚动，层次分明 |
| 5 | 基础操控 | 移动、跳跃、互动、攀爬 |
| 6 | 谜题 1 | 隐形平台——信息差指引通过 |
| 7 | 谜题 2 | 单向开关——非对称操作打开门 |
| 8 | 谜题 3 | 协作推箱子——物理差异协作 |
| 9 | 存档点 | 双方同时到达触发存档，显示 MVP 完成画面 |
| 10 | 死亡重生 | 坠落后从最近检查点重生 |

## 10. 实现优先级

```
P0 (必须):
├── 双人联机 (房间码匹配 + 位置同步)
├── 双世界渲染 (同逻辑层 + 异视觉皮肤)
├── 视差背景 (5 层)
├── 玩家操控 (移动/跳跃/互动)
├── 3 个谜题 + 1 个存档点
└── 死亡/重生

P1 (应该):
├── 简易文字聊天 (游戏内沟通备用)
├── 音效差异 (乐观方听到鸟鸣，悲观方听到风声)
└── 角色动画 (行走/跳跃/站立)

P2 (可以):
├── 开场动画/过渡
├── 存档点视觉特效
└── 背景环境动画 (飘叶/飘灰)
```
